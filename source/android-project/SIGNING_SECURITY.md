# Безопасное хранение release-подписи

Материалы release-подписи критичны для будущих обновлений приложения:

```text
stalker-pda-release.jks
release-password.txt
```

## Правила

- Никогда не публикуйте эти файлы в GitHub, облачном публичном архиве или чате.
- Храните минимум две зашифрованные резервные копии в разных физических местах.
- Не переименовывайте alias `stalkerpda`, если хотите использовать существующий скрипт без изменений.
- Не меняйте `applicationId` для обновлений release-версии.
- Увеличивайте `versionCode` перед каждым следующим выпуском.
- Не встраивайте пароль в manifest, JavaScript, smali или APK.
- Для CI используйте secret storage платформы и создавайте временный password-файл только на время сборки.

Потеря keystore или пароля означает невозможность подписать обновление тем же сертификатом.
