# КПК Зоны — отчёт исправления v5

## Зафиксированная ошибка v4

На физическом Android-устройстве ART отклонил класс `LocalWebViewClient`:

```text
java.lang.VerifyError
Can flow through to move-exception
```

Причина находилась в методе `shouldOverrideUrlLoading(WebView, WebResourceRequest)`. После защищённого `try` обычное выполнение могло последовательно попасть на первую инструкцию обработчика `move-exception`. В DEX `move-exception` разрешён только как точка входа обработчика исключения.

## Исправление первопричины

- Вынесено открытие внешних ссылок в отдельный метод `openExternal(Uri)`.
- После успешного `startActivity()` добавлен безусловный переход мимо обработчика.
- Обработчик начинается с `move-exception` и больше не достижим обычным потоком управления.
- Добавлен автоматический аудит всех DEX/Smali-обработчиков, который проверяет:
  - первую инструкцию каждого catch;
  - отсутствие обычного fall-through;
  - отсутствие `goto`/`if` в catch-вход;
  - корректность try-диапазонов;
  - границы регистров;
  - соответствие инструкций возврата сигнатурам;
  - корректное расположение `move-result`.

## Дополнительные исправления полного аудита

1. **WebView-навигация**
   - локальным считается только `https://localhost` без нестандартного порта или с портом 443;
   - добавлены обе сигнатуры `shouldOverrideUrlLoading`;
   - во внешнее приложение разрешено передавать только `http`, `https` и `mailto`;
   - `file:`, `content:`, `javascript:`, `intent:` и неизвестные схемы блокируются.

2. **Камера**
   - WebView получает только ресурс `VIDEO_CAPTURE`;
   - запросы микрофона, DRM и неизвестных WebView-ресурсов не подтверждаются;
   - после системного разрешения повторно выдаётся только видеоресурс.

3. **GPS**
   - исправлена проверка origin: больше нет уязвимого сопоставления по префиксу;
   - поддерживается как точное, так и приблизительное разрешение Android;
   - результат считается разрешённым, если выдано хотя бы одно из `FINE`/`COARSE`.

4. **Выбор файлов**
   - добавлена защита от `null` callback/params;
   - при ошибке выбора callback обязательно завершается значением `null`;
   - перехватываются безопасно все обычные исключения запуска системного picker.

5. **Совместимость обновления**
   - release сохраняет пакет `by.smorhon.stalkerpda.v3`;
   - release сохраняет прежний сертификат;
   - `versionCode` увеличен с `10003` до `10004`;
   - debug использует отдельный пакет `by.smorhon.stalkerpda.v5.debug` и не конфликтует со старыми debug-сборками.

## Выполненные проверки

- `node --check` для всех трёх встроенных JavaScript-блоков;
- анализ HTML-ID и всех `byId()`-ссылок;
- отсутствие внешних runtime-ресурсов;
- отсутствие `eval`, `new Function`, присваиваний `innerHTML`, `document.write` и отладочного кода;
- исходный Smali-аудит: 7 классов, 36 методов, 6 exception handlers;
- бинарный DEX-аудит debug и release;
- проверка, что каждый бинарный catch начинается с `move-exception` и не имеет обычного входа;
- полная декомпиляция всех собственных DEX-классов через Androguard DAD;
- повторное декодирование обеих APK через APKTool;
- ZIP CRC и структура APK;
- четырёхбайтовое выравнивание несжатых записей;
- APK Signature Scheme v2 для debug/release;
- совпадение release-сертификата v4 и v5;
- DOM-smoke: карта, жёсткие границы, масштабирование, метка, заметка, QR, адаптивность 320–1440 px;
- camera-smoke: запуск синтетического видеопотока и обязательная остановка трека при закрытии;
- отсутствие необработанных ошибок страницы в доступных DOM-тестах.

## Остаточное ограничение

В среде сборки нет Android-эмулятора и физического устройства, поэтому финальный v5 APK не был установлен и запущен здесь на ART. Ошибка со скриншота устранена непосредственно в DEX и покрыта двумя независимыми проверками: исходного Smali и собранного бинарного DEX. Окончательная полевая проверка выполняется на телефоне.
