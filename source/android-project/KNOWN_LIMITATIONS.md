# Известные ограничения

1. В доступной сборочной среде не было Android-эмулятора, физического Android-устройства и `adb`. Поэтому APK не устанавливался и не запускался здесь на реальном Android runtime. Это ограничение явно отражено в отчёте тестирования.
2. Камера была проверена в Chromium с синтетическим MediaStream, но не через физический модуль телефона.
3. GPS проверен браузерной имитацией координат, но точность реального приёмника и системный Android-диалог разрешения требуют полевого теста.
4. Системный выбор файлов, `ACTION_CREATE_DOCUMENT`, Android runtime permissions и кнопка «Назад» проверены статически по native-коду и успешной сборкой, но не динамически на Android.
5. Данные обычной браузерной версии не мигрируют в APK автоматически. Используйте экспорт/импорт JSON.
6. Debug и release имеют разные package ID и отдельные хранилища.
7. APK подписан схемой v2. Поддерживаемый минимум приложения — Android 7.0 / API 24.
8. Работа зависит от установленного Android System WebView/Chrome WebView. На сильно устаревшем или отключённом WebView современные camera/QR API могут вести себя иначе.
