# Отчёт о тестировании Android-сборки

Дата: 2026-07-13  
Версия: 1.0.0 (`versionCode 10000`)

## Итог

APK-файлы фактически собраны, подписаны и проверены библиотекой apksig. Однофайловое SPA прошло автоматизированный браузерный регрессионный тест, включая карту, метки, заметки, QR, сохранение, адаптивность, синтетическую камеру и имитацию GPS.

В среде выполнения отсутствовали Android Emulator, физическое устройство и `adb`. Поэтому установка APK, запуск в Android System WebView, системные permission-диалоги, Android file picker, Android back stack и Logcat **не были фактически проверены на Android**. Эти пункты нельзя честно пометить как пройденные.

## Среда

- OpenJDK/Javac: 21.0.10
- Node.js: 22.16.0
- npm: 10.9.2
- Python: 3.13.5

- Браузерный движок теста: Chromium headless

## Исправленные проблемы упаковки

1. `file://` заменён защищённым локальным origin, чтобы Web APIs камеры и GPS не блокировались происхождением документа.
2. Экспорт через браузерный download заменён Android document picker при наличии bridge.
3. Добавлена нативная выдача runtime permissions вместо безусловного разрешения WebView.
4. Разрешения выдаются только origin `localhost`.
5. Внешние URL не загружаются внутри WebView.
6. Отключены cleartext, file URL access, mixed content, WebView debugging, third-party cookies и backup.
7. Debug и release разделены package ID.
8. Перекрывающиеся permission callbacks очищаются, чтобы не возникала гонка запросов.
9. Системный back сначала передаётся внутренней логике SPA.

## Остаточные риски

Главный риск — отсутствие фактического Android runtime-теста. Перед распространением необходимо установить `app-release.apk` на минимум два устройства/API, пройти камеру, GPS, file picker, экспорт, поворот экрана, сворачивание/возврат, back navigation и проверить Logcat.
