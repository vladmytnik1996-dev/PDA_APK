# Журнал изменений Android-версии

## 1.0.0

### Добавлено

- Нативная Android Activity для запуска SPA из APK.
- Защищённый локальный origin `https://localhost` без сетевого сервера.
- Runtime-запрос разрешения камеры.
- Runtime-запрос точной/примерной геолокации.
- Системный выбор карты, QR-изображения и JSON-файлов.
- Системное сохранение JSON-резервной копии.
- Обработка системной кнопки «Назад» с учётом внутренних окон и режимов SPA.
- Сохранение WebView при смене ориентации и изменении размера экрана.
- Отдельный package ID для debug-сборки.
- Подписанная release-сборка и проверка схемы подписи.

### Минимальные изменения веб-приложения

- Добавлен вызов Android-моста для экспорта JSON с сохранением браузерного fallback.
- Добавлена функция `window.pdaHandleAndroidBack()`.
- Остальная логика карты, жёстких границ, GPS-калибровки, меток, заметок и QR-обмена сохранена.

### Исправлено при упаковке

- Исключён `file://`-origin, из-за которого камера и GPS часто блокируются.
- Исключён внешний localhost-сервер: документ обслуживается из APK.
- Запрещён mixed content в WebView.
- Отклоняются запросы камеры/геолокации от origin, отличного от localhost.
- Старый незавершённый запрос геолокации корректно отклоняется перед новым.
- Исправлено освобождение ссылок WebView/permission callbacks при паузе и уничтожении Activity.
- Экспорт данных больше не зависит от браузерного Download API внутри WebView.

## 1.0.2 — install/runtime hotfix

- Исправлена обязательная 4-байтовая alignment `resources.arsc` для target API 30+.
- Возвращена APK Signature Scheme v2 вместо несовместимой v1-only подписи.
- Исправлены catch-handler DEX: добавлены обязательные `move-exception` и разделены обычные/exception переходы.
- Новые applicationId `by.smorhon.stalkerpda.v3` и `.v3.debug`.
## 1.0.5 / fixed v6

- включён immersive sticky fullscreen;
- скрыты status bar и navigation bar;
- fullscreen восстанавливается после фокуса и resume;
- добавлена поддержка display cutout;
- добавлена верхняя safe-area в веб-интерфейс.
